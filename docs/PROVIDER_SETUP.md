# 模型服务配置

## 推荐流程

1. 首次引导或“模型与平台”中选择官方预设。
2. 核对 Base URL，并按服务商控制台填写当前有效 Model ID。
3. 输入 API Key 并保存。
4. 主动执行最小连接测试。
5. 创建或更新 ModelProfile，填写上下文长度、最大输出和能力。
6. 在“模型策略”中按任务选择模型，或生成默认策略。

预设只是静态起点。服务商可能更改 Model ID、价格、地区限制与能力；请以各自官方控制台和文档为准。

## 支持范围

当前真实模型链路保证 OpenAI Chat Completions Compatible：OpenAI、DeepSeek、Moonshot/Kimi、智谱、阿里云百炼、Gemini OpenAI Compatible 与小米 MiMo 的兼容端点。未实现各厂商原生协议和自动模型列表。

## 常见错误

| 错误 | 建议 |
| --- | --- |
| Key 未配置或无效 | 重新保存凭据并测试连接 |
| 余额不足 | 在服务商控制台充值或更换模型 |
| 模型不存在/无权限 | 核对 Model ID 与账号权限 |
| 429 限流 | 稍后重试或降低并发/输出长度 |
| Base URL 错误 | 使用预设后核对区域和兼容端点 |
| 超时或网络失败 | 检查网络、代理和服务商状态 |
| 图片不受支持 | 选择声明 `imageInput` 的 ModelProfile；纯文本模型不会收到图片 |

## 价格

成本页不会内置或猜测实时价格。只有你为 ModelProfile 配置输入/输出每百万 Token 价格，并且服务商实际返回 Usage 时，应用才显示估算费用；否则显示“未知”或“未配置价格”。
