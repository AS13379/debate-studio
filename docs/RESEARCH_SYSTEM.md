# 研究与证据系统

## 可见性模型

| 区域 | 创建者 | 可见者 | 用途 |
| --- | --- | --- | --- |
| 公共资源池 | 主持人 | 全体 | 概念、范围、少量来源入口与事实边界 |
| 私有研究空间 | 正方或反方 | 所属方与用户 | 研究计划、查询、资料、笔记、暂定主张 |
| 公开证据桌 | 任一方/主持人 | 全体 | 正式发言中引用、可质疑和核验的证据 |

研究阶段生成的内容必须附带 `visibility` 与 `ownerParticipantId`。任何供另一方模型调用的上下文构造器都只读取公共资源、公开证据和该方自己的私有研究。用户可在 UI 查看记录，但不应将一方私有记录自动投递给另一模型。

## 研究流程

1. 主持人建立有限公共资源池，不输出完整论证。
2. 双方各自产出可见的研究计划与目标。
3. （后续）各方通过独立 `SearchSession` 请求检索；第一阶段可由用户提交文字/图片资料。
4. 各方对来源作出评价、形成笔记与暂定主张。
5. 在正式发言引用某资料时，显式发布至证据桌，获得稳定编号。
6. 对方提出质疑或反证；主持人更新证据状态并记录说明。

## 资产与转换记录

`ResearchAsset` 记录原始文件/URL、MIME、来源与处理谱系。第一阶段只支持 `text` 与 `image`：文本保存内容或安全引用，图片保存在应用数据目录并记录原始 MIME。尚未实现的 OCR、PDF 提取、转录、关键帧与视觉描述仅以 `AssetTransformation` 接口保留，UI 必须明确标为未实现。

## 证据状态

`unverified` 是默认状态；`supported`、`disputed`、`outdated`、`inaccessible`、`misleading`、`rejected` 都必须有记录人、时间和说明。状态表达当前核验结论而不是观点胜负；保留历史，不能静默删除不利证据。

## 第一阶段界线

实现人工提交资料、资产元数据、公开证据表、引用关联和基本显示。定义搜索接口与研究实体，但不接入搜索 API、正文抓取、自动核验或附件转换流水线。
