# 分阶段开发计划

## Phase 0：本轮完成

完成需求收敛、架构、协议、研究/流程文档和类型草案。没有应用初始化、真实 API 或精细 UI。

## Phase 1：最小可运行骨架

1. 创建 Electron + React + TypeScript + Vite 工程，配置 Vitest。
2. 建立 main/preload/renderer 边界、基础中文路由和 Zod IPC 合约。
3. 初始化 SQLite、迁移和仓储；建立安全的应用数据目录与资产目录。
4. 实现领域状态机及其单元测试。
5. 实现 MockAdapter、TurnRunner、流事件持久化与取消语义。

验收：可通过 MockAdapter 完成、暂停/继续、停止、失败重试并恢复查看一场固定辩论。

## Phase 2：连接与记录

1. 实现连接/模型配置、凭据存储抽象、连接测试和手填 Model ID。
2. 实现 OpenAIChatAdapter 的文本流、基础图片和错误标准化。
3. 实现实时辩论页、历史/详情页、Token 与费用基础显示。
4. 实现 JSON/Markdown 导出及密钥泄漏测试。

验收：用户可用一个兼容端点运行固定流程，所有轮次可保存、错误可理解、导出无密钥。

## Phase 3：研究与控制完善

1. 实现公共资源池、私有研究记录、公开证据桌与引用显示。
2. 实现用户提问、提交证据、跳过、强制下一阶段和强制总结。
3. 强化崩溃恢复、流中断显示、资产导入校验与集成测试。

验收：研究可见性不泄漏，用户控制命令可追溯，图片处理路径被明确记录。

## 后续（不纳入第一阶段）

搜索工具、原生协议、PDF/音频/视频转换、自动核验、上下文压缩、无限辩论周期和更完整价格策略，均在真实使用验证第一阶段后再排期。

## 建议的下一项单一开发任务

初始化 Electron + React + TypeScript + Vite 项目，并先实现不含真实 API 的 `DebateEngine` 状态机与 Vitest 测试。这项工作可尽早验证核心流程，而不会锁定供应商或 UI 设计。
