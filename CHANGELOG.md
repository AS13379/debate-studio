# Debate Studio 更新日志

本文件记录 Debate Studio 各公开版本的重要变化。版本号遵循 [Semantic Versioning](https://semver.org/)。

## [0.5.4] - 2026-07-22

### 社区更新安装可靠性

- 修复旧应用退出后 macOS 单实例锁尚未释放，导致新版本只启动一次便退出并被误判失败的问题。
- 安装助手现在会等待旧进程完全结束，留出系统稳定时间，并在两分钟健康确认窗口内重复尝试启动新版本。
- 安装成功后清理旧应用备份与更新缓存；回滚时清理残留的 pending 状态，避免旧版本继续显示错误的更新进度。
- 增加真实 shell 安装集成测试，覆盖重试后成功替换以及启动失败自动回滚。

## [0.5.3] - 2026-07-22

### 导出保存与历史菜单

- 导出 Markdown 或 HTML 前先显示 macOS 原生保存窗口，由用户明确选择文件名和保存位置；取消选择时不创建后台任务。
- 选择位置后提示任务已创建，后台生成完成后再次显示导出成功和最终保存位置。
- 导出提示增加右侧关闭按钮，错误提示和历史详情中的导出状态也可主动关闭。
- 修复辩论历史卡片启用渲染优化后，管理菜单被卡片内容或相邻卡片遮挡的问题。

## [0.5.2] - 2026-07-21

### 社区更新校验与自动下载

- 修复 Electron 运行时把应用框架符号链接的无害解压警告误判成校验失败的问题，并保留已通过签名与哈希校验的更新包供重试和排查。
- 将解压、目录安全、Bundle ID 和版本读取拆分为独立校验步骤；失败界面显示安全的步骤代码，不再统一吞成未知错误。
- 校验失败后显示真实缓存占用，可重新下载并校验、在 Finder 中查看缓存包或改用手动 DMG。
- 重启应用后仍可在 Finder 中定位已下载的项目签名更新包；本地预发布版高于 GitHub 最新版时不再误报降级攻击。
- 增加“发现新版后自动下载”开关；默认关闭，开启后会在检查到新版本时自动下载并校验，但安装仍需用户确认。
- 自动更新不读取或修改 SQLite、模型配置、API Key、辩论及研究数据。

## [0.5.1] - 2026-07-21

### 研究防循环与证据兜底

- 将原先单一的“工具预算”改为分阶段防循环策略：探索阶段限制重复决策、搜索、网页读取和正文总量，成果整理阶段继续允许保存笔记、形成主张、发布证据和结束研究。
- 增加“快速、标准、深入”三档研究预设，额度用于控制等待时间和重复工具调用，不再以节省 API 费用为目标。
- 连续多轮没有新资料或达到探索深度后自动进入成果整理，不再让模型继续无效搜索。
- 自动模式结束研究前会发布已读取正文且被模型明确建议发布的可靠来源，避免资料已经收集却没有进入公开证据桌。
- 研究界面改为显示模型决策、研究动作、搜索和读页进度，并兼容旧版本已保存的研究预设。
- 新增 migration v16，持久化研究阶段、防循环计数和完成原因，重启后仍能解释研究为何收束。

## [0.5.0] - 2026-07-21

### 社区自动更新

- 使用 Debate Studio 项目 Ed25519 签名的社区更新包替代未签名 macOS 上不可用的 Squirrel 安装流程。
- 下载后严格校验签名、SHA256、文件大小、版本、arm64 架构与 `com.leander.debatestudio` Bundle ID。
- 安装时先备份当前应用；新版本无法启动会自动恢复并重新打开旧版本。
- 设置页显示下载、校验、安装、回滚和缓存状态，可在 Finder 中查看更新包、清理缓存或转到 GitHub 手动下载 DMG。
- 更新只替换 `.app` 程序文件，不读取或修改 SQLite、模型配置、Prompt、API Key、辩论和研究数据。

### 升级说明

- v0.4.9 及更早版本需要最后一次手动下载 v0.5.0 DMG 并覆盖安装。
- 从 v0.5.0 开始，后续版本可以通过应用内项目签名更新完成升级。
- 当前构建仍未使用 Apple Developer ID 签名和公证；项目签名用于验证更新来源与完整性，不会绕过 Gatekeeper。

## [0.4.9] - 2026-07-21

### 官方模型能力预设

- 根据各平台官方模型资料补齐 OpenAI、Moonshot / Kimi、DeepSeek、智谱、MiMo、阿里云百炼和 Gemini 的输入模态与运行能力预设。
- 选择模型后自动勾选文本、图片、PDF、音频、视频、流式输出、深度思考、工具调用、内置搜索和结构化输出能力；自定义 Model ID 仍可手动调整。
- 实时 `/models` 响应缺少能力字段时，会自动合并应用内置的官方预设；带日期的官方模型快照也能继承对应基础模型能力。
- 更新 DeepSeek V4、GLM-5.1、MiMo V2.5、Qwen 3.7、Kimi K3/K2.6 和 Gemini 3.5 的上下文及主要能力信息。

## [0.4.8] - 2026-07-21

### 百炼多模型兼容与等待反馈

- 补充百炼工作空间中的 GLM-5.2、DeepSeek 和 MiMo 离线模型能力信息，不再把百炼仅视为千问模型平台。
- GLM-5.2 连接测试明确关闭深度思考并使用安全的最小输出额度，避免健康连接因思考预热而长时间无反馈。
- 模型编辑器会应用模型目录返回的能力，并允许明确开关深度思考；长思考模型会提前提示首字等待。
- Provider 日志新增响应头、首个 SSE 事件和总耗时节点，不记录模型正文、API Key 或凭据引用。
- 所有长操作在等待超过 8 秒后说明当前可能处于排队、冷启动或模型思考阶段，超过 30 秒仍可主动停止。

## [0.4.7] - 2026-07-21

### 辩论上下文修复

- 修复流程内裁决没有收到正反方正式发言、因而错误报告“未收到任何辩论内容”的问题。
- 交叉质询、反驳和总结现在会读取此前已完成的公开发言，不再仅依赖研究资料和公开证据。
- 正式发言上下文只包含已完成的公开 Turn，排除私有研究阶段、失败片段和未完成内容。
- 长辩论上下文采用有界截取并同时保留开头与结尾，避免丢失开篇或总结。

## [0.4.2] - 2026-07-20

### Electron 与 LAN Web 视觉统一

- Electron 与桌面 LAN Web 共享同一套工作台外壳、品牌区、页面标题、创建模式、运行控制、参与角色和发言卡片组件。
- LAN Web 直接继承桌面 Renderer 的页面宽度、间距、字体、颜色、按钮和卡片设计令牌，不再维护第二套近似样式。
- 统一辩论列表、新建辩论和实时辩论的结构顺序；已完成或已停止的辩论收起为紧凑运行状态栏。
- 移动端仅在 768px 以下应用底部导航、单列卡片和触控布局，桌面浏览器保持与 Electron 一致。
- 保留现有 LAN 数据访问适配、安全边界和全部业务行为，不增加新的远程功能。

## [0.4.1] - 2026-07-20

### 局域网 Web 工作台

- 增加可选的局域网 Web 控制台，可在手机、平板和桌面浏览器中查看历史、创建辩论并控制实时运行。
- 支持 AI 自动规划、AI 辅助完善和手动创建，复用桌面端模型路由与辩论配置逻辑。
- 支持研究与证据、评分与复盘、成本摘要、Markdown/HTML 导出下载，以及图片和 PDF 安全上传。
- 实时页面通过 WebSocket 同步阶段、流式发言和运行状态，并提供启动、暂停、继续与停止控制。

### 访问模式与安全

- 设置中可选择“仅本机访问”或“可信局域网访问”；个人局域网模式无需密码和设备配对。
- Web 端不接触 API Key、`credentialRef`、SQLite、本机绝对路径或凭据存储。
- 保留 Host/Origin/CSRF 校验、请求大小限制、基础限流、上传文件签名检查和安全响应 DTO。
- 局域网模式持续提示仅在个人可信网络使用，不建议在公共 Wi-Fi 开启。

### 体验与稳定性

- 针对 iPhone、Android、iPad 和桌面浏览器提供响应式布局、触控尺寸与长文本换行优化。
- 增加长辩论分页快照、旧 Turn 按需加载、流式更新节流和断线后快照恢复。
- 修复 WebSocket 路由注册、移动端横向溢出和 LAN 服务启停恢复问题。

### macOS

- 提供 Apple Silicon（arm64）DMG。
- 当前公开 DMG 未经过 Apple Developer ID 签名和 Apple 公证，macOS 首次打开时可能需要在“系统设置 → 隐私与安全性”中手动允许。

## [0.4.0] - 2026-07-19

### Kimi / Moonshot 思考模型

- 完整支持 Kimi K3 与 K2.x 思考内容的流式展示和多轮工具调用续传，减少长时间思考被误认为卡死的情况。
- 修复 `reasoning_content` 丢失、工具参数分片组装不完整、思考额度耗尽及 Kimi 特有参数不兼容问题。
- Kimi K3 请求使用平台允许的最大完成额度，避免长研究在生成正文或工具参数前被截断。

### 运行与恢复

- 正式辩论、研究、辩题规划、裁判、复盘和图片分析默认不再设置应用侧输出 Token 限制。
- 完善长运行心跳、实时活动说明、强制跳过当前阶段、失败轮次重试及部分文本保留。
- 结构化区分输出截断、工具 JSON 错误、连接问题和服务商错误，同时保留用户重试控制权。

### 界面与隐私

- 修复浅色模式下侧边栏、边框和辅助文字仍使用深色样式的问题。
- 思考内容仅在当前运行界面瞬时展示，不写入 SQLite、日志、诊断报告或导出文件。

## [0.3.6] - 2026-07-19

### 修复与改进

- 取消正式辩论、研究、辩题规划、裁判、复盘和视觉分析的默认应用侧输出 Token 限制；仅保留服务商自身的模型边界。
- Kimi K3 请求使用平台允许的最大 `max_completion_tokens` 1,048,576，避免 131,072 的服务商默认额度在长研究中提前耗尽。
- 模型配置中的“最大输出”明确为能力信息，不再默认作为实际请求限额。

## [0.3.5] - 2026-07-19

### 修复与改进

- 修复 Kimi K3 被应用侧强制限制为 16,000 输出 Token，导致思考完成前即被截断的问题。
- Kimi K3 以及开启思考的 K2.5/K2.6 不再发送应用侧 `max_tokens`，使用服务商的模型默认额度；旧的 800/16,000 Token 本地配置不再截断请求。
- 新建 Kimi K3 默认配置更新为官方 131,072 Token 默认输出额度，并校正模型目录的最大输出信息。

## [0.3.4] - 2026-07-19

### 修复与改进

- 兼容展示旧版本已经保存的 Kimi `EMPTY_RESPONSE / finish_reason=length` 失败记录，将其明确解释为可重试的模型输出上限问题。
- 历史失败 Turn 保持原样且继续展示原始技术详情；重试会创建使用新 Kimi 输出配置的 Turn，不覆盖旧记录。

## [0.3.3] - 2026-07-19

### 修复与改进

- 修复 Kimi 思考模型因较小 `max_tokens` 被思考内容提前耗尽，继而产生空回复或残缺工具 JSON 的问题；思考模式现在使用官方建议的安全输出下限。
- 将 `finish_reason=length` 识别为可恢复的输出截断错误，保留已收到的思考内容与正文，并允许重试当前 Turn。
- 错误分类不再剥夺用户控制权：即使系统建议先检查配置，失败或中断的 Turn 仍可选择“仍然重试”。
- 修复浅色模式下侧边栏仍保持深色的问题，并补齐窄窗口侧边栏分隔线的深浅色切换。

## [0.3.2] - 2026-07-19

### 修复与改进

- 为所有 OpenAI Compatible 模型增加独立的实时思考内容通道：服务商返回 `reasoning_content` 时可在正式辩论、自主研究和辩题规划中展开查看。
- 思考内容与公开回答严格分流；未返回可见思考文本时，界面仍持续显示运行时长和活动状态。
- 修复流式研究工具调用的增量参数组装，使 Kimi、DeepSeek、GLM、MiMo、Qwen 等思考模型能在多步工具链中继续运行。
- 按官方协议区分 Kimi K3、K2.7 Code 和 K2.5/K2.6 的思考参数，并为 DeepSeek、智谱、MiMo、千问与 Gemini 配置兼容的思考开关。
- 保留 Gemini 工具调用中的 `thought_signature`，避免多步函数调用因缺少服务商续传状态而失败。

### 本地数据与隐私

- 思考文本仅在当前 Renderer 内存中瞬时展示，并在 IPC 边界自动脱敏；不进入 Turn、SQLite、日志、历史事件、诊断报告或导出文件。
- Gemini 可能返回思考摘要而非完整内部推理，界面统一标注为“服务商返回的思考内容 / 摘要”。

## [0.3.1] - 2026-07-19

### 修复与改进

- 修复 Kimi K3、Kimi K2.x 与其他 OpenAI Compatible 思考模型在多步工具调用中丢失 `reasoning_content`、导致研究阶段 HTTP 400 或停滞的问题。
- 工具链现在只在当前内存请求链中原样续传服务商推理上下文；完成、失败或中断后立即释放。
- 研究期间优先展示服务商放在普通 `content` 中的可见阶段说明，并定期显示等待时长，避免长思考被误认为空转。
- 更新推理上下文错误的中文说明，使其可重试并明确本地隐私边界。

### 本地数据与隐私

- `reasoning_content` 不进入 Turn、SQLite、日志、IPC、导出或 Renderer；界面只显示公开 `content` 与不含推理正文的活动状态。

## [0.3.0] - 2026-07-18

### 新增

- 增加服务商模型目录与在线模型发现，可从当前模型列表选择，也继续支持自定义 Model ID。
- 扩展 Moonshot / Kimi 模型支持，并在服务商配置中提供平台、接入文档和官方定价入口。
- 增加内置官方定价目录、来源与核验日期；成本统计可自动匹配已知模型并保留手动覆盖能力。
- 保存模型连接后自动执行最小连接测试，不再要求用户返回页面寻找测试按钮。

- 研究阶段支持有界的独立搜索与网页读取并发，并为主持人公共资源池采用更快的研究预算。

### 修复与改进

- 修复 DeepSeek 工具调用与思考模式续写不兼容造成的超时和停滞。
- 增加长思考模型提示、研究心跳和当前活动说明，减少长请求被误认为无响应的情况。
- 改进研究取消、失败恢复、部分结果保留及中文错误呈现。
- 修复首次引导中服务商平台链接错误显示为 Kimi 的问题。

### 本地数据与隐私

- API Key 继续仅存于本机加密凭据存储，并只在主进程请求 Header 中使用。
- 模型目录、定价与进度事件均不向 Renderer 返回 API Key 或凭据引用。
- 本版本不改变现有 SQLite 数据路径；覆盖安装后会继续使用原有模型配置和本地辩论记录。

### macOS

- 提供 Apple Silicon（arm64）DMG。
- 当前公开 DMG 未经过 Apple Developer ID 签名和 Apple 公证，macOS 首次打开时可能需要在“系统设置 → 隐私与安全性”中手动允许。

## [0.2.4] - 2026-07-16

### 新增

- 本地 AI 辩论工作台，支持创建、运行、暂停、继续、停止和恢复辩论。
- OpenAI Compatible 模型连接，可为主持人、正方、反方和裁判配置不同模型。
- AI 辩题规划，支持自动规划、辅助完善和完全手动三种创建方式。
- 自动研究与证据系统，支持隔离的正反方研究空间、公共资源池、网页研究和证据状态历史。
- Prompt Studio，支持 Prompt 版本创建、比较、回滚和调用记录。
- 结构化裁判评分、赛后复盘和辩论质量分析。
- 多模态基础能力，包括图片与 PDF 资产管理、缩略图以及 Vision 模型能力检查。
- Markdown 与 HTML 辩论导出、历史管理、诊断日志、数据库备份与恢复。

### 本地数据与隐私

- 辩论、研究、证据、设置和运行记录保存在本地 SQLite 数据库中。
- API Key 仅保存在本机加密凭据存储中，不写入 SQLite、日志、导出文件或 Renderer。
- 用户主动调用模型或搜索服务时，凭据只发送给用户选择的第三方服务商；Debate Studio 不提供自有云服务，也不会把密钥或用户数据上传给项目方。
- Renderer 启用沙箱和上下文隔离，并通过类型安全的白名单 IPC 访问主进程能力。

### macOS

- 提供 Apple Silicon（arm64）DMG 构建。
- 修复开发环境与打包应用的 Dock 图标，并提供可复现的透明 PNG/ICNS 生成流程。
- 当前公开 DMG 未经过 Apple Developer ID 签名和 Apple 公证。首次打开时，macOS 可能要求用户在“系统设置 → 隐私与安全性”中手动允许。
